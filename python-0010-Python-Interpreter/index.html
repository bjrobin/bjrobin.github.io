那么，问题来了，刚才不是讲，如果不使用默认编码，那么文件的第一 行要写成特殊的注释。否则可能会出现乱码。
那为什么，没有写特殊注释，也能输出中文，没有看到乱码呢？
原来这是IDLE做的手脚。
当我们的源文件里面的代码全部都是英文时，IDLE把源文件保存为ANSI 编码的格式。
由于没有额外的特殊的注释，IDLE以为这是英文的程序，会按照iso8859-1编码来读取。
在我们的中文操作系统，ANSI 编码就是GB2312编码。GB2312编码表示英文26个小写英文字母还有26个大写英文字母与iso8859-1编码的表示方法是一样的，所以不用加上特殊的注释也没问题。
但如果源码中有中文，IDLE在保存问件时，把源文件自动保存为UTF-8 编码的格式。这样也没问题。
但是，如果源码中有中文，没有特别的注释，文件却保存为ANSI 编码，IDLE还以为是英文的文件呢，会按照iso8859-1编码读取文件，iso8859-1编码本身就没法表示中文， 这就有问题了。

总结：

本节课，知识点比较复杂也比较不好理解，请记住如下几点：

  1. 保存到“我的电脑”里的程序文件。可以是“ANSI”编码，也可以是“UTF-8”编码。
  2. “ANSI”编码可以表示中文，“UTF-8”编码也可以表示中文。
  3. 还有一种编码：“iso-8859-1”，只能表示英文。
  4. 如果是“ANSI”编码，需要添加特殊的注释。
  5. 如果是“ANSI”编码，并且没有特殊的注释，IDLE会以为只有英文，就会按照“iso-8859-1”来读取文件。这样如果有中文，就会出现错误，有可能出现乱码。
  6. 统一的简单的解决办法：永远保存为“UTF-8”。这样IDLE有办法知道这是“UTF-8”，不会出现错误，没有乱码。
